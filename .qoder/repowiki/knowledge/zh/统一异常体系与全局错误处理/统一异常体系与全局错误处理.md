本仓库采用「分层异常类型 + 集中错误码枚举 + 全局异常处理器」的三层架构，贯穿 DDD 各层（adaptor/application/domain/infrastructure），保证错误在边界处被统一捕获并返回一致的 ResultDO。

## 1. 系统概览

- **异常基类**：`BizException` 是所有业务异常的根，携带 `code` 与 `msg`，支持通过 `ErrorCodeEnum` 构造。
- **领域细分异常**：`AggregateException`、`RepositoryException`、`EventProcessException`、`AdaptorException` 分别对应聚合根、仓储、事件处理、适配层的特定失败场景，均继承自 `BizException`。
- **错误码中心**：`ErrorCodeEnum` 收敛全部错误码，按通用、持久层、认证、用户、角色权限、字典、文件等分组，每个条目包含 code 与默认文案。
- **全局兜底**：`GlobalExceptionHandler` 使用 `@RestControllerAdvice` 拦截 Sa-Token 鉴权异常、参数解析异常、404 以及未捕获的 `Exception`，统一转为 `ResultDO`，不向客户端泄露堆栈。

## 2. 关键文件与职责

- `common/exception/BizException.java` — 业务异常基类，封装 code/msg 及 cause。
- `common/exception/AggregateException.java` — 聚合根状态/不变量校验失败。
- `common/exception/RepositoryException.java` — 持久化读写失败。
- `common/exception/EventProcessException.java` — 领域事件处理失败。
- `common/exception/AdaptorException.java` — 适配器层输入校验或外部调用失败。
- `common/result/ErrorCodeEnum.java` — 全项目唯一错误码来源，禁止散落字符串字面量。
- `adaptor/common/GlobalExceptionHandler.java` — 全局异常转换，覆盖 Sa-Token 未登录/无权限、JSON 反序列化、参数类型不匹配、404 及兜底 Exception。

## 3. 架构与约定

- **抛出位置**：domain/application 层遇到业务规则违反时抛 `AggregateException`；infrastructure 层抛 `RepositoryException`；adaptor 层抛 `AdaptorException`；事件监听器抛 `EventProcessException`。
- **错误码引用**：所有异常构造必须通过 `ErrorCodeEnum`，避免硬编码错误码。
- **响应包装**：成功路径返回 `ResultDO`，失败路径由 `GlobalExceptionHandler` 将异常转换为 `ResultDO.buildFailResult(...)`。
- **安全异常隔离**：Sa-Token 的 `NotLoginException`、`NotRoleException`、`NotPermissionException` 由全局处理器单独映射为 401/403，不混入业务异常流。
- **日志策略**：鉴权/参数类异常用 warn 级别记录上下文信息，兜底异常用 error 记录完整堆栈。

## 4. 开发者应遵循的规则

1. **只抛自定义异常**：不要在业务代码中直接 return `ResultDO.fail(...)`，而是抛对应的 `*Exception`，让上层或全局处理器统一包装。
2. **始终使用 ErrorCodeEnum**：新增错误码先在 `ErrorCodeEnum` 中声明，再在异常构造中引用。
3. **按层选择异常类型**：聚合规则 → `AggregateException`；数据库操作 → `RepositoryException`；事件处理 → `EventProcessException`；HTTP 适配 → `AdaptorException`。
4. **不要吞掉异常**：catch 后应转换为业务异常向上抛出，或在明确可恢复场景下记录日志并继续执行。
5. **不在 Controller 中 try-catch 业务异常**：交由 `GlobalExceptionHandler` 统一处理，保持控制器简洁。
6. **对外消息可读**：`ErrorCodeEnum` 的默认文案面向最终用户，如需更具体提示可在构造异常时覆写 msg。
