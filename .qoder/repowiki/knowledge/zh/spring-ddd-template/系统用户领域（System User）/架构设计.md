本模块按 DDD 分层将用户领域拆为三个协作子包：
- `client/system/user/` 定义跨进程契约（`UserAppService`、`UserQueryAppService` 及请求/响应 DTO），作为唯一对外入口；
- `application/system/user/` 提供具体实现（`UserAppServiceImpl`、`UserQueryAppServiceImpl`），负责参数校验、DTO↔领域模型转换（`UserAssembler`）、事件发布（`UserCreatedListener`）以及与其他领域服务的编排；
- `adaptor/system/user/input/UserController` 仅做 HTTP 适配，把 Spring MVC 请求转换为 `client` 层的 DTO 并委托给应用服务。
三者通过 `client` 包中的接口与 DTO 形成单向依赖：adaptor → client 接口 → application 实现，禁止反向引用，从而保证用户领域对上层框架的解耦。
