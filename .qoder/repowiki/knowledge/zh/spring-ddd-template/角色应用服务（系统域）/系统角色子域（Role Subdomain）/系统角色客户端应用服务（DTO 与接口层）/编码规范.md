- 每个写操作/读操作都成对出现 RequestDTO 与 ResponseDTO，命名遵循 `<Verb><Noun>RequestDTO` / `<Verb><Noun>ResponseDTO` 模式，分别放在 req/ 与 res/ 子包。
- 所有 DTO 继承 `BaseDto`，声明 `@Getter/@Setter/@ToString` 与 `serialVersionUID`，保持序列化与样板代码一致。
- 请求参数校验在 RequestDTO 内覆写 `check()` 方法，使用正则常量 + `ResultDO.buildFailResult(ErrorCodeEnum.PARAM_ERROR, ...)` 返回结构化错误，而非抛异常。
- 对外枚举（如 `RoleStatusEnum`）独立于内部枚举定义，并提供 `getByCode(Integer)` 静态工厂方法，避免 client 层直接依赖领域内部类型。
- 应用服务接口按 CQRS 拆分为 Cmd/Query 两个独立接口，分别继承 `ApplicationCmdService` 与 `ApplicationQueryService`，方法签名统一以 `ResultDO<ResponseDTO>` 作为返回值。
