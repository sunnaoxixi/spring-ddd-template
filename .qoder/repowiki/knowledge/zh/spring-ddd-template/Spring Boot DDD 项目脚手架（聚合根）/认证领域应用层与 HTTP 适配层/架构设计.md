本模块遵循 DDD 分层：
- `adaptor.auth.input.AuthController` 作为 Spring MVC Input Adaptor，仅做参数绑定与注解（@Tag/@Operation/@OperLog），无业务逻辑，直接委托给 client 接口。
- `client.auth` 定义对外契约：`AuthAppService`（写模式，继承 `ApplicationCmdService`）暴露 login/register/logout；`AuthQueryAppService`（读模式，继承 `ApplicationQueryService`）暴露 getLoginUserInfo；请求/响应 DTO 按 req/res 子包组织。
- `application.auth.scenario` 提供两个场景实现：`AuthAppServiceImpl` 编排写流程（参数自校验 → `LoginAttemptLimiter` 防爆破 → `AuthDomainService.authenticate` / `UserDomainService.createUser` → Sa-Token 签发/注销 → 填充 Token-Session 附加信息 → 发布 `LoginLogEvent` 异步落库）；`AuthQueryAppServiceImpl` 从 StpUtil 取登录 ID 后查 `UserRepository` + `RoleRepository` 组装响应。
- `application.auth.assembler.AuthAssembler` 使用 MapStruct（componentModel=spring）负责 RequestDTO/ResponseDTO 与领域 Param/Aggregate 之间的转换，枚举通过 @Named 辅助方法映射。
依赖方向单向向下：adaptor → client 接口 → application 实现 → domain 服务/仓库；Sa-Token 调用收敛在应用层，领域不感知会话技术细节。
