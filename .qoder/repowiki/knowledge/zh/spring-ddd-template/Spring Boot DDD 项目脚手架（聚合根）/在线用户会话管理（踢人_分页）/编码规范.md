- 应用服务接口按 CQRS 拆分为写模式（继承 `ApplicationCmdService`）与读模式（继承 `ApplicationQueryService`），分别位于独立接口文件。
- 所有方法入口先调用 `requestDTO.check()` 进行参数自校验，失败时通过 `ResultDO.buildFailResult` 快速返回。
- 异常统一由 try-catch 捕获并记录 `log.error`，再返回 `ErrorCodeEnum.SYSTEM_ERROR`，避免异常泄漏到上层。
- HTTP 控制器仅做参数转换与转发，业务逻辑全部下沉至应用层实现类，控制器本身不包含任何业务判断。
