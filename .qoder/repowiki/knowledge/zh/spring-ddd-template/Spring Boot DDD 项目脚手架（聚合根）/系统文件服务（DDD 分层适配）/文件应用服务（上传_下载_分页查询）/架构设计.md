本模块位于 application/system/file 包，遵循 DDD 分层：
- `FileStorage` 是 outAdaptor 接口（定义在 application 层），通过依赖倒置将本地磁盘、OSS 等具体存储实现隔离到 adaptor 层，所有方法统一返回 `ResultDO<T>` 不抛异常。
- `scenario/FileAppServiceImpl` 负责写模式场景编排：参数自校验 → 调用 FileStorage.store 写入物理文件 → 调用领域服务 `FileDomainService.createFile` 登记元数据；若元数据登记失败则尽力回滚物理文件。
- `scenario/FileQueryAppServiceImpl` 负责读模式：经 `FileRepository` 查聚合根，再经 FileStorage.read 取物理内容，最后由 Assembler 组装响应。
- `assembler/FileAssembler` 使用 MapStruct（`@Mapper(componentModel="spring")`）声明式地将 RequestDTO/ResponseDTO 与领域对象（`FileAggregate`/`FileEntity`/Param）互相转换，避免在 Service 中散落映射逻辑。
- 操作人信息一律从 `CurrentUserContext.getUserId()` 注入，保证审计字段来源一致。
