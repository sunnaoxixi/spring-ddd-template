- HTTP 控制器仅做参数绑定与 DTO 组装，业务逻辑全部委托给 `client.*.LogQueryAppService`，禁止在 Controller 中编写业务判断。
- 应用服务方法遵循「参数自校验 → 构建 PageQuery → 调用领域 Repository → Assembler 转 ResponseDTO」四步模板，异常统一捕获并返回 `ErrorCodeEnum.SYSTEM_ERROR`。
- 写日志采用事件驱动：领域发布 `*LogEvent`，应用层以 `@Async @EventListener` 监听并异步落库，失败只打日志不抛异常，保证主流程不受影响。
- 对象转换集中放在 `LogAssembler` 中，MapStruct 声明式映射 + `default` 方法处理空聚合根与空列表的兜底逻辑。
- 对外契约集中在 `client.system.log` 包（接口 + req/res/model DTO），adaptor 与 application 均通过该包通信，避免直接引用对方内部类。
