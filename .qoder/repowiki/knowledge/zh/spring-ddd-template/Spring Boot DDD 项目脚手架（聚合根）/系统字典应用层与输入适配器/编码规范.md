- 应用服务方法统一采用「请求自校验 → 调用领域服务 → 组装响应」三段式模板，异常捕获后返回 `ResultDO.buildFailResult(ErrorCodeEnum.SYSTEM_ERROR)`。
- 所有入参 DTO 均暴露 `check()` 方法用于参数自校验，应用层在执行业务前优先检查其成功性。
- 写操作通过 `CurrentUserContext.getUserId()` 注入当前操作人 ID，并由 Assembler 以 `@Context Long operatorId` 传入领域 Param。
- client 层的状态码与领域 model 枚举通过 `DictStatusEnum.getByCode()/getCode()` 在 Assembler 的 default 方法中双向转换，避免在业务代码散落。
- HTTP 端点按资源维度分组于 `/api/system/dicts/{types,data}`，写接口统一加 `@SaCheckPermission("system:dict:write")`，读接口加 `system:dict:read`。
