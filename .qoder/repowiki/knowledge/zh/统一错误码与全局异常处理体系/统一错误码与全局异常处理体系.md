本仓库采用「集中式错误码枚举 + 分层异常类型 + 全局异常处理器」的三层架构来管理错误，贯穿 DDD 六层。

## 1. 系统/方法
- 错误码收敛：ErrorCodeEnum 作为全局唯一入口，按通用、持久层、认证、用户、角色权限、字典、文件等域分组，每个条目包含 code 与默认 defaultMsg，禁止散落字符串字面量。
- 异常类型族：以 BizException 为根（携带 code 字段），派生出领域/仓储/适配器等语义化子类 AggregateException、RepositoryException、AdaptorException、EventProcessException，用于在对应层抛出业务错误。
- 统一结果对象：ResultDO<T> 提供 buildSuccessResult / buildFailResult(ErrorCodeEnum|String) 工厂方法，各层方法统一返回该对象而非抛异常；注释明确禁止向调用方直接抛出异常。
- 全局异常兜底：GlobalExceptionHandler 使用 @RestControllerAdvice 捕获 Sa-Token 鉴权异常（未登录/无角色/无权限）、请求体解析异常、参数类型不匹配、404 以及所有未预期 Exception，统一转为 ResultDO 并记录日志，不向外泄露堆栈。

## 2. 关键文件
- common/result/ErrorCodeEnum.java — 全局错误码定义
- common/model/BaseResult.java / common/result/ResultDO.java — 统一响应封装
- common/exception/BizException.java 及 AggregateException、RepositoryException、AdaptorException、EventProcessException — 分层异常族
- adaptor/common/GlobalExceptionHandler.java — 全局异常拦截与转换

## 3. 架构与约定
- 正常路径：Domain/App/Repository 层通过返回 ResultDO 表达成功/失败，由上层组合判断，避免异常流控制业务逻辑。
- 异常路径：当出现不可恢复的业务错误时，在对应层抛出带 ErrorCodeEnum 的 BizException 子类，由上层 catch 后包装成 ResultDO 返回。
- 边界兜底：进入 Controller 之前的框架级异常（鉴权、反序列化、路由）由 GlobalExceptionHandler 统一处理，确保对外只暴露 ResultDO。
- 安全原则：全局异常处理器仅记录错误日志，不将堆栈或内部细节返回给客户端。

## 4. 开发者规则
1. 新增错误码一律在 ErrorCodeEnum 中追加，禁止硬编码字符串。
2. 业务层优先返回 ResultDO；仅在不可恢复的错误场景抛出 BizException 及其子类。
3. 不要自行 try/catch 后再抛 RuntimeException，应转换为 ResultDO 或由 GlobalExceptionHandler 兜底。
4. 不要在日志中输出敏感信息；全局异常处理器已负责记录堆栈。
5. 对 Sa-Token 相关异常无需单独处理，交由 GlobalExceptionHandler 统一映射到 NOT_LOGIN / NO_PERMISSION。
