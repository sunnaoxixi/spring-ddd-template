本模块是 DDD 分层中的 client 层，仅暴露对外契约，不包含实现：
- 通过 `UserAppService`（继承 `ApplicationCmdService`）定义写操作（create/update/changeStatus/delete），通过 `UserQueryAppService`（继承 `ApplicationQueryService`）定义读操作（getUserDetail/queryUserPage），形成明确的 CQRS 边界。
- 每个方法以成对的 `req/*RequestDTO` + `res/*ResponseDTO` 作为入参出参，由上层 application 层的 Assembler 负责与内部 domain model 转换；client 层禁止依赖内部 model 层，枚举也独立定义在 `enums/` 下（如 `UserStatusEnum`）。
- `model/UserDTO` 是跨多个 ResponseDTO 复用的轻量聚合视图对象，不含密码等敏感字段。
- 所有 DTO 继承 `BaseDto` 并实现 `check()` 完成参数校验，返回 `ResultDO<Void>`，校验失败统一走 `ErrorCodeEnum.PARAM_ERROR`。
