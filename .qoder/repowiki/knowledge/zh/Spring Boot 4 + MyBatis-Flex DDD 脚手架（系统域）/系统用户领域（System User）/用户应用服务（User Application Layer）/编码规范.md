- 应用方法统一采用「参数自校验 → 调用领域服务 → 组装响应」三段式结构，校验失败直接返回 `ResultDO.buildFailResult`，异常由外层 try/catch 捕获并转为 `ErrorCodeEnum.SYSTEM_ERROR`。
- 请求对象通过自身 `check()` 方法完成入参校验，应用层只判断 `ResultDO.isSuccess()` 而不重复编写校验逻辑。
- 操作人标识一律从 `CurrentUserContext.getUserId()` 获取，经 Assembler 的 `@Context Long operatorId` 注入到领域 Param，不在 Controller 或领域层透传。
- 跨领域数据填充（如角色标识）在应用层通过额外 Repository（`RoleRepository`）批量查询后回填聚合根，避免 N+1 查询。
- MapStruct 中所有涉及 client/model 枚举的转换都集中在 UserAssembler 的 default 方法或 `@Named` 辅助方法中，保持接口层与领域层类型解耦。
