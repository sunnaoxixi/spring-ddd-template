本模块位于 application/system/user 包下，遵循 DDD 分层：
- scenario/ 提供两个 Spring Service：`UserAppServiceImpl`（写模式，实现 `client.system.user.UserAppService`）负责创建/更新/状态变更/删除的用户场景编排；`UserQueryAppServiceImpl`（读模式，实现 `UserQueryAppService`）负责详情与分页查询。
- assembler/UserAssembler.java 基于 MapStruct（`@Mapper(componentModel="spring")`）声明 DTO ↔ Param/Aggregate 的转换规则，枚举值在 mapper 层统一做 int↔enum 映射。
- listener/UserCreatedListener.java 通过 `@Async + @EventListener` 异步消费 `domain.system.user.event.UserCreatedEvent`，示例仅记录日志，失败不阻断主流程。
依赖方向单向向下：application → domain（`UserDomainService`、`UserRepository`、`RoleRepository`），并通过 `CurrentUserContext` 注入操作人上下文；写路径禁用/删除后在应用层调用 Sa-Token 的 `StpUtil.kickout` 强制下线会话，避免侵入领域模型。
