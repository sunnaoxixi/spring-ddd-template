项目采用严格的 DDD 分层：adaptor（Controller + GlobalExceptionHandler + OperLogAspect）→ application（Assembler + Scenario AppService）→ domain（Aggregate + DomainService + Repository 接口）→ infrastructure（RepositoryImpl + Converter + MySQL Mapper/PO），client 层定义跨进程 DTO 契约，common 层提供 BaseAggregate/BaseEntity/BaseResult 等共享基类与 SaToken/MyBatis-Flex/OpenAPI 配置。

- 启动入口 `SpringDddTemplateApplication` 扫描 `com.sunnao.spring.ddd.template` 包，通过 `@MapperScan` 注册 MyBatis-Flex Mapper，SaToken 在 `infrastructure.auth.StpInterfaceImpl` 中实现权限校验。
- 每个子域（auth、system.dict/file/log/role/user）遵循相同目录镜像结构，领域事件通过 `DomainEventPublisher` → `SpringDomainEventPublisher` 桥接 Spring ApplicationEvent，由 application 层 Listener（如 LoginLogListener、OperLogListener、UserCreatedListener）消费。
- 数据库变更由 Flyway 管理，`db/migration/V*.sql` 按版本初始化 sys_user、RBAC、oper_log、dict、file、login_log 表。
- Maven Wrapper (`mvnw`) 驱动构建，`docker-compose.yaml` 编排依赖的 MySQL/Redis 环境。
