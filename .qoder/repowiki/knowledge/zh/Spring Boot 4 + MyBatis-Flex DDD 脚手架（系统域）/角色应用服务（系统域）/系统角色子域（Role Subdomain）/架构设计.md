本模块以 DDD 的「client / application / adaptor」三层切分角色子域：
- `client.system.role` 定义 RoleAppService / RoleQueryAppService 接口及全部 req/res DTO，作为跨进程/跨层的稳定契约，被其他子系统通过 RPC 或本地调用消费。
- `application.system.role` 实现上述两个接口，由 `assembler.RoleAssembler` 负责领域模型与 DTO 之间的转换，并编排参数校验与领域服务调用；写读场景分别落在 `scenario/RoleAppServiceImpl` 与 `RoleQueryAppServiceImpl`。
- `adaptor.system.role.input.RoleController` 仅做 HTTP 适配，把请求 DTO 转为应用层入参后委托给应用服务。
三者之间唯一的耦合点是 `client` 包中的接口与 DTO 契约，adaptor 与 application 均依赖它，从而保证上层控制器与应用实现可独立演进。
