本模块按 DDD 适配器-应用层-领域依赖方向组织：
- `adaptor.system.log.input.LogController` 作为 HTTP Input Adaptor，仅做参数绑定、权限校验（Sa-Token `@SaCheckPermission("system:log:read")`）与 DTO 组装，调用 `client.system.log.LogQueryAppService`。
- `client.system.log` 包暴露读模式应用服务接口及请求/响应 DTO，是跨进程/模块的契约边界；实现位于 `application.system.log.scenario.LogQueryAppServiceImpl`，职责为参数自校验 → 通过 `LogAssembler` 转为领域 `PageQuery<OperLogQuery|LoginLogQuery>` → 调用领域 Repository 分页查询 → 再经 Assembler 转回 ResponseDTO。
- `application.system.log.assembler.LogAssembler` 基于 MapStruct（`@Mapper(componentModel="spring")`）负责 RequestDTO↔领域 Query、Aggregate↔DTO 的双向转换。
- 写路径不暴露 API：`OperLogListener` 与 `LoginLogListener` 使用 `@Async @EventListener` 异步消费 `OperLogEvent` / `LoginLogEvent`，构造对应 Aggregate 后落库，失败仅记录日志不影响主流程。
- 依赖方向单向：adaptor → client 接口 → application 实现 → domain repository/aggregate/event，本模块不直接依赖外部子系统。
