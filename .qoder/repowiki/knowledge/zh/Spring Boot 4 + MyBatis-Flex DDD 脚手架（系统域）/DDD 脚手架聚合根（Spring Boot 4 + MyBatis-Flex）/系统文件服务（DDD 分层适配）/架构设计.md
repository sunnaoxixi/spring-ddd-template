本模块遵循 DDD 分层契约：client 层仅暴露 FileAppService/FileQueryAppService 两个应用接口及对应 DTO，作为跨进程调用边界；application 层实现 FileAppServiceImpl/FileQueryAppServiceImpl，通过 FileStorage 抽象与领域层交互，并借助 FileAssembler 在 DTO 与领域模型间转换；adaptor 层由 FileController 接收 HTTP 请求并委托给 application 接口。三层之间以接口和 DTO 为唯一耦合点，adaptor 不依赖 application 实现，application 不感知 HTTP 细节，形成单向依赖链 client → application → adaptor。
