- HTTP 控制器只负责参数绑定与注解装饰，所有业务编排下沉到 application 层的 AppService 实现类。
- 应用服务按 CQRS 拆分为写模式（继承 ApplicationCmdService）与读模式（继承 ApplicationQueryService）两个独立接口及实现。
- 领域对象与外部 DTO 的转换集中在 MapStruct Assembler 中，通过 default 方法与 @Named 辅助方法处理复杂映射。
- 跨领域协作（如角色标识）通过 Repository 查询后回填聚合根实体字段，再由 Assembler 提取为 ResponseDTO。
- 易失败的外部副作用（登录日志事件发布、Token-Session 写入）包裹 try-catch 并仅记录日志，确保不影响主业务流程。
