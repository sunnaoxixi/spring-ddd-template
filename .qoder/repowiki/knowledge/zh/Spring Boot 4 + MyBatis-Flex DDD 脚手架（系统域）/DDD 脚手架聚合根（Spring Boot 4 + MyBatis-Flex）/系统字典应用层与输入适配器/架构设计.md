本模块遵循 DDD 分层：
- `adaptor.system.dict.input.DictController` 作为 Input Adaptor，仅做参数绑定、权限校验（Sa-Token `system:dict:read|write`）与操作日志（`@OperLog`），无业务逻辑。
- `client.system.dict.*` 定义对外暴露的应用服务接口（`DictAppService` 写模式继承 `ApplicationCmdService`，`DictQueryAppService` 读模式继承 `ApplicationQueryService`）以及 Request/Response/Model DTO 与枚举，是应用层对外的契约边界。
- `application.system.dict.scenario` 下的 `DictAppServiceImpl` / `DictQueryAppServiceImpl` 负责场景编排：先调用 `requestDTO.check()` 自校验，再经 `DictAssembler` 转为领域 Param 调用 `DictDomainService`，最后组装响应；查询路径直接通过 `DictRepository` 获取聚合根并转 DTO。
- `application.system.dict.assembler.DictAssembler` 使用 MapStruct（`@Mapper(componentModel="spring")`）集中处理 DTO↔Param 与 Aggregate→DTO 的映射，并在 default 方法中完成 client 状态码与 model 枚举的双向转换。
依赖方向单向：adaptor → client 接口 → application 实现 → domain 服务/仓储，不反向依赖。
