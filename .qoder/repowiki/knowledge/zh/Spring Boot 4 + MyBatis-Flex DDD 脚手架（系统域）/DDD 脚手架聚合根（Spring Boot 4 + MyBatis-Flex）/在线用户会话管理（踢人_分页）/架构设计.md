模块遵循 DDD 适配层 + 应用层的分层结构：
- `adaptor.system.online.input.OnlineController` 作为 HTTP Input Adaptor，仅做参数绑定、鉴权（`@SaCheckRole("admin")`）与操作日志（`@OperLog`），无业务逻辑。
- `client.system.online.*` 定义对外暴露的应用服务接口（`OnlineAppService` 写模式继承 `ApplicationCmdService`，`OnlineQueryAppService` 读模式继承 `ApplicationQueryService`）及请求/响应 DTO，是适配器与应用层之间的契约边界。
- `application.system.online.scenario` 下按场景拆分实现：`OnlineAppServiceImpl` 封装 `StpUtil.kickout*` 完成踢人；`OnlineQueryAppServiceImpl` 全量扫描 Sa-Token Redis token 键、过滤无效会话后在内存分页，返回 `OnlineUserDTO` 聚合的会话信息。
- 依赖方向单向：Controller → client 接口 → application 实现 → Sa-Token SDK，不直接依赖领域层或持久化仓储。
