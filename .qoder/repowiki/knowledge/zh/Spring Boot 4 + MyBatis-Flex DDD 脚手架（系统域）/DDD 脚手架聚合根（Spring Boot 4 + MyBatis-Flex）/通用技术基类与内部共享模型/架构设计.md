本模块由两个并列包组成，职责严格分离：
- `common`（技术抽象层）：不承载业务概念，仅暴露各层共享的技术基础设施。其中 `model/` 定义 DDD 基类（`BaseEntity`、`BasePO`、`BaseDto`、`BaseParam`、`BaseResult`、`Aggregate`、`Repository` 等），`exception/` 提供 `BizException` 及聚合/仓储/事件相关异常，`service/` 用空标记接口 `ApplicationCmdService` / `ApplicationQueryService` / `DomainService` 作为应用服务契约；`context/` 通过 `CurrentUserContext` 包装 Sa-Token 登录态供审计监听器使用；`config/` 集中装配 Sa-Token 路由拦截、Security、OpenAPI、MybatisFlex、异步线程池；`lock/` 基于 `LevelLock` 抽象提供 JVM 级与 Redis 级分布式锁实现；`result/` 的 `ResultDO` 是跨层统一返回值封装。
- `model`（内部共享模型层）：存放跨模块复用的纯业务模型，当前以 `system/*` 下的状态枚举为主（`DictStatusEnum`、`FileStorageTypeEnum`、`RoleStatusEnum`、`UserStatusEnum`），遵循 `{名称}Enum` 命名约定。
依赖方向：domain/application/adaptor/infrastructure 可依赖本模块，client 层禁止依赖 model（防止扩散到外部调用方）。
